# Building_HTTP-1.1_Protocol_From_TCP

## Project Abstract

### This project is a hands-on implementation of an HTTP server, built from the ground up using low-level networking primitives. It explores how raw byte streams are handled over TCP, how HTTP requests are parsed and interpreted, and how well-formed HTTP responses are constructed and delivered to clients.

### Throughout the project, the server evolves from processing basic request lines to handling complex features like headers, bodies, chunked transfer encoding, and binary data transmission.

- Key Topics & Learning Milestones
- Read and interpret incoming byte streams over TCP
- Parse HTTP request lines and headers manually
- Handle request bodies with and without content-length
- Construct and send valid HTTP responses
- Implement chunked transfer encoding
- Handle binary data and accommodate multiple HTTP versions

### This project is an exercise in building a deeper, protocol-level understanding of how HTTP servers operate — beyond frameworks and libraries — focusing on how data flows from socket to response, and how low-level components power the modern web.
